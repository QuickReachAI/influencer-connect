Authentication Security Overhaul Security
In plain English

Previously, anyone who could guess a user's ID number could pretend to be that user. Now, every login cookie is cryptographically signed — like adding a tamper-proof seal to an envelope. If someone tries to forge a cookie, the system rejects it immediately.

Technical details
  • Added HMAC-SHA256 cookie signing using crypto.createHmac with NEXTAUTH_SECRET
  • Cookie value format changed from userId to userId.hmacSignature
  • Added signCookie(), verifyCookie(), and getAuthUserId() to auth helpers
  • Constant-time comparison to prevent timing attacks
  • Updated 27 API route files to verify signatures before trusting user identity

Deal State Machine Implemented Business Logic
In plain English

Deals go through many stages — from draft, to negotiation, to script approval, to payment, to production, to delivery, to completion. Previously, there was no central system enforcing which stage can follow which. A deal could theoretically jump from "draft" straight to "completed." Now there's a strict state machine that only allows valid transitions and logs every change.

Technical details
  • Added VALID_TRANSITIONS map covering all 12 DealStatus enum values
  • Added transitionDealStatus(dealId, targetStatus, actorId) with transaction + audit logging
  • Refactored lockDeal, acceptLock, rejectLock to use the state machine
  • Uses Prisma's DealStatus enum for type safety
DRAFT → LOCKED → SCRIPT_PENDING → SCRIPT_APPROVED
→ PAYMENT_50_PENDING → PRODUCTION → DELIVERY_PENDING
→ PAYMENT_100_PENDING → COMPLETED

Any active state → DISPUTED or CANCELLED

Escrow Approval Gate Fixed Bug Fix
In plain English

Previously, when a creator uploaded their deliverables, the system automatically started a 2-day countdown to release payment — even if the brand hadn't reviewed the work yet. Now, the brand must explicitly approve the deliverables before the payment countdown begins.

Technical details
  • handleFileUploadTrigger now transitions to DELIVERY_PENDING (was incorrectly going to PAYMENT_100_PENDING)
  • Added approveDeliverables(dealId, brandId) as the explicit brand approval gate
  • Only approveDeliverables can trigger DELIVERY_PENDING → PAYMENT_100_PENDING
  • Payment route now checks deal.status !== 'PAYMENT_100_PENDING' instead of !deal.filesUploaded
Creator uploads files → DELIVERY_PENDING
Brand reviews & approves → PAYMENT_100_PENDING
Brand pays → T+2 timer → COMPLETED

Cursor-Based Pagination Feature
In plain English

Instead of loading all campaigns at once, the API now returns them in batches of 20 with a bookmark to get the next batch. This is faster, more reliable, and handles new campaigns being added without skipping or duplicating results.

Technical details
  • Replaced page/pageSize with cursor (last campaign ID) and limit (default 20, max 50)
  • Uses Prisma's native cursor/skip pattern — fetches limit + 1 to determine hasMore
  • Response: { data: Campaign[], nextCursor: string | null, hasMore: boolean }

Mock Database Fallback Removed Bug Fix
In plain English

The app used to silently fall back to a fake in-memory database when no real database was configured. This meant the app could appear to work fine while actually not connecting to anything real — hiding bugs that would only surface in production. Now it fails immediately with a clear error message.

Technical details
  • Rewrote lib/prisma.ts as a standard Prisma singleton
  • Throws Error('DATABASE_URL environment variable is required') if missing
  • Uses globalThis caching pattern for Next.js hot-reload compatibility
